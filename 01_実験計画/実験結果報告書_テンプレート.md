## 1. 実験概要

本実験は、申請書「着想経緯 (c)」に記載した「LLMによる回路認識・生成における入力形式依存性」を定量的に検証するために実施した。

### 検証対象の主張
1. **H1（形式依存性）**: 回路を「構造体の集合＋接続情報（構造化JSON）」として記述すると、アスキーアートや自然言語記述と比べて、LLMの回路トポロジ認識・生成の精度が向上する。
2. **H2（非対称性）**: LLMは電力工学の専門知識（数値計算・解析）を有するが、空間的トポロジ認識（どの素子がどこに配置されているか）は依然として課題が残る。

## 7. 考察：申請課題への示唆

本実験の結果は、申請書で主張する「CIM着想の構造体データ表現によって汎用LLMを電力回路設計に活用できる」という方向性を以下の点で裏付ける。

1. **構造体JSON形式の優位性**（H1が支持された場合）：TyphoonHILのPython API `hil.set_component_property()` は構造体データ形式で回路を操作するため、本実験で示した「JSON形式での精度向上」は、AIエージェント→HIL操作の実現可能性を直接支持する。

2. **専門知識の存在確認**（H2が支持された場合）：LLMが電力工学の数値計算（共振周波数・デューティ比・変調指数等）を正確に解くことができる一方で、空間的トポロジ認識には課題があることを示す。これは「空間認識の代わりに構造体データを使う」という本研究のアプローチの根拠となる。

3. **スケーラビリティへの示唆**：より複雑な回路（C4: 3相VSI）で形式依存性が顕著であれば、実際のインバータ設計において構造体表現の重要性がより高くなることを示す。

## 付録：実験で使用した回路とプロンプト例

### A. テスト回路一覧

#### C1: 直列RLC回路
- 素子: V1(10V), R1(100Ω), L1(10mH), C1(1μF)
- 正解: 共振周波数 503.3Hz, インピーダンス 100Ω, Q値 0.316

#### C2: H-bridgeインバータ
- 素子: VDC(48V), Q1-Q4(MOSFET×4), Rload(10Ω)
- 正解: 出力電圧ピーク 48V, 電流 4.8A

#### C3: Buckコンバータ
- 素子: Vin(24V), Q1(MOSFET), D1(ダイオード), L1(100μH), C1(100μF), Rload(5Ω)
- 正解: 出力 12V (D=0.5), 電流リプル 0.6A

#### C4: 3相VSIインバータ
- 素子: VDC(600V), CDC(1000μF), Q1-Q6(IGBT×6), Rload×3(10Ω)
- 正解: 線間電圧実効値 300V, 相電圧実効値 173.2V

### B. 入力形式の例（C1, 直列RLC回路）

#### Form-A（アスキーアート）
```
     N1      N2      N3
N0 ──┤V1├──┬──┤R1├──┬──┤L1├──┬──┤C1├──┐
           │         │         │         │
GND ───────┴─────────┴─────────┴─────────┘
```

#### Form-B（自然言語記述）
> V1(10V)、R1(100Ω)、L1(10mH)、C1(1μF)を直列接続。
> V1正極→N1→R1→N2→L1→N3→C1→N0(GND)の順で接続。

#### Form-C（構造体JSON）
```json
{
  "components": [
    {"id": "V1", "type": "VoltageSource", "value": 10, "unit": "V"},
    {"id": "R1", "type": "Resistor", "value": 100, "unit": "Ω"},
    {"id": "L1", "type": "Inductor", "value": 10, "unit": "mH"},
    {"id": "C1", "type": "Capacitor", "value": 1, "unit": "μF"}
  ],
  "connections": [
    {"from": "V1+", "to": "N1"},
    {"from": "N1",  "to": "R1"},
    {"from": "R1",  "to": "N2"},
    {"from": "N2",  "to": "L1"},
    {"from": "L1",  "to": "N3"},
    {"from": "N3",  "to": "C1"},
    {"from": "C1",  "to": "N0"}
  ]
}
```
