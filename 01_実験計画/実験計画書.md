## 1. 研究目的と検証仮説

### 背景
申請者はこれまで、LLMを用いた回路認識・生成において以下の3つの入力形式を試行してきた。

| 条件 | 入力形式 | 観察された傾向 |
|------|----------|----------------|
| **Form-A** | アスキーアート（文字回路図） | 空間トポロジ解釈困難、精度に限界 |
| **Form-B** | コンポーネントメタデータ＋自然言語接続記述 | LLMの既知パターンを引き出す形で一定成果 |
| **Form-C** | 構造体JSON（コンポーネント配列＋接続配列） | テキスト・画像入力より明確に精度向上 |

### 検証仮説（Research Hypothesis）
> **H1**: 回路を「構造体の集合＋接続情報（構造化JSON）」として記述すると、アスキーアートや自然言語記述と比べて、LLMの回路トポロジ認識・回路生成の精度が統計的に有意に向上する。
>
> **H2**: LLMは電力工学の専門知識（パラメータ計算・制御理論）を有するが、空間的トポロジ認識（どの素子がどこに配置されているか）は依然として課題が残る。

### 2.1 対象回路（4種類）

| 回路ID | 回路名 | 難易度 | 素子数 | ノード数 | 電力工学的意義 |
|--------|--------|--------|--------|----------|----------------|
| C1 | 直列RLC回路 | ★☆☆☆ | 3 | 4 | 基礎フィルタ / 共振現象 |
| C2 | H-bridgeインバータ | ★★★☆ | 6 | 6 | PWMインバータの基本トポロジ |
| C3 | Buckコンバータ | ★★☆☆ | 5 | 5 | 降圧DC-DC変換器 |
| C4 | 3相VSIインバータ | ★★★★ | 9 | 8 | AC駆動システムの中核 |

### 2.2 タスク設計（3種類）

#### Task T1: トポロジ認識タスク
- **入力**: 3形式のうちの1つで回路を記述
- **出力**: コンポーネント一覧（種類・値）＋接続関係（ネットリスト）
- **評価**: コンポーネント識別率・接続精度・完全一致率

#### Task T2: 解析タスク（空間認識 vs 専門知識の分離）
- **入力**: 3形式のうちの1つで回路を記述
- **出力**: 電気的特性の計算（共振周波数・インピーダンス・デューティ比等）
- **評価**: 数値正確度（±5%以内を正解とする）
- **目的**: H2の検証（専門知識は持つが空間認識は苦手という非対称性）

#### Task T3: 生成タスク
- **入力**: 仕様文（「共振周波数100Hzの直列RLC回路を設計せよ」等）＋出力形式指定
- **出力**: 3形式のうちの1つの形式で回路を生成
- **評価**: 仕様充足率・構造的妥当性・数値的妥当性

## 3. 評価指標定義

### 3.1 コンポーネント識別率（CR: Component Recognition Rate）
```
CR = (正しく識別したコンポーネント数) / (正解の全コンポーネント数)
```
- 種類が合っている: 1点
- 種類かつ値（±10%以内）が合っている: 2点
- 最大スコア = コンポーネント数 × 2

### 3.2 接続正確度（CA: Connection Accuracy）
```
CA = (正しく識別した接続エッジ数) / (正解の全接続エッジ数)
```
- 無向グラフとして比較（A-B の接続 = B-A の接続）

### 3.3 トポロジ完全一致率（TE: Topology Exact-match Rate）
```
TE = 1 if (CR = 1.0 AND CA = 1.0) else 0
3回試行の平均でスコア化
```

### 3.4 数値正確度（NA: Numerical Accuracy）
```
NA = 1 if |computed - answer| / |answer| < 0.05 else 0
計算できなかった場合は NA = 0
```

### 3.5 生成妥当性スコア（VS: Validity Score）
```
VS = w1 * 仕様充足率 + w2 * 構造的妥当性 + w3 * 数値妥当性
w1 = 0.4, w2 = 0.4, w3 = 0.2
```

## 4. 統計解析計画

- **一元配置分散分析（One-way ANOVA）**: 3形式間のCR・CA・TEの差
- **Tukey HSD多重比較**: Form-A vs C、Form-B vs C の有意差検定
- **有意水準**: α = 0.05
- **効果量**: Cohen's d で実践的有意性を評価

## 5. 予想される結果と申請書への記載方針

| 評価指標 | Form-A(ASCII) | Form-B(自然言語) | Form-C(JSON) | 予想差異 |
|----------|---------------|-----------------|--------------|----------|
| CR（認識率） | 0.50–0.70 | 0.65–0.80 | 0.80–0.95 | C > B > A |
| CA（接続精度） | 0.30–0.55 | 0.55–0.70 | 0.75–0.90 | C > B >> A |
| TE（完全一致） | 0.10–0.30 | 0.25–0.45 | 0.50–0.80 | C >> B > A |
| NA（解析精度） | 0.60–0.80 | 0.70–0.85 | 0.75–0.90 | 差小（専門知識依存） |

- T1/T3での形式依存性が顕著 → H1の支持
- T2での形式依存性が小 → H2の支持（「専門知識はあるが空間認識が苦手」）
